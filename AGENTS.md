# AGENTS.md — 项目开发约定

1. **每行代码都需要增加注释 what，有必要时需要增加 why。**
   每一行或每一个逻辑块都要有中文注释说明它在做什么（what）。如果背后的选择不是显而易见的（比如为什么选这个方案、为什么避开某个坑），必须写上原因（why）。

2. **不过度设计。**
   优先选择最简单、最直接的实现方式。不提前引入抽象、模式、配置、泛化，除非当前确实需要。等有了重复或扩展的真实需求，再重构。

3. **不确定就问。**
   任何不确定的都需要用提问工具向我提问，以获得准确的信息。不要猜测接口、需求或实现细节。

4. **最小化改动。**
   修复 bug 或加功能时，只改动最少的必要代码。不顺手重构、不顺手格式化、不顺手调整无关的布局或命名。改一行能解决的事，不改三行。

5. **沉淀有效信息。**
   开发过程中发现的对项目有价值的经验、问题根源、配置技巧，都要存到 `docs/` 下对应的分类目录里。
   - 问题/Bug 记录 → `docs/bugs/`
   - 有用的技巧/最佳实践 → `docs/tips/`
   - 架构决策 → `docs/decisions/`
   格式不限，关键是要写清楚 what（发生了什么）和 why（根因/为什么这么解决），让以后的自己和 AI 能快速看懂。
